# Alumni App 用户端改造实施计划

## 阶段一：基础层

1. 重建 `theme.scss` 的颜色、字号、间距、圆角、阴影、按钮、表单、状态和动效变量。
2. 优化 `Layout.vue`，统一安全区、页面宽度、背景与进入动效。
3. 新增页面标题、反馈状态、骨架屏和弹层等基础组件。
4. 更新 `pages.json` 的导航栏与 TabBar 颜色。

## 阶段二：请求与共享逻辑

1. 重构请求层，统一鉴权、错误信息和登录失效处理。
2. 修正前端契约中的 `phase` 字段。
3. 新增验证码倒计时和可编辑列表组合式逻辑。
4. 去除重复鉴权和散落的深拷贝逻辑。

## 阶段三：页面改造

1. 登录、注册与验证码交互。
2. 资讯真实空状态。
3. 活动列表、卡片、加载和分页状态。
4. 活动详情、元信息、报名弹层与固定操作区。
5. 签到、签到结果和签到管理。
6. 个人信息、教育经历和工作经历。

## 阶段四：验证

1. 运行 TypeScript 类型检查。
2. 运行 H5 与微信小程序构建。
3. 启动 H5 预览，检查关键页面、控制台错误、响应式布局和动效降级。
4. 根据验证结果修复问题并复查工作区差异。
