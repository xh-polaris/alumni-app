# Alumni App 用户端视觉与体验改造设计

## 1. 目标

全面优化 `alumni-app` 微信小程序用户端，包括登录、注册、资讯、活动列表、活动详情、报名、签到、签到管理、个人信息、教育经历和工作经历。

本次改造的优先级是：

1. 界面信息清晰、操作路径直接。
2. 形成统一且有辨识度的校友平台视觉语言。
3. 提升加载、空状态、异常、编辑和提交过程的反馈质量。
4. 在不调整后端协议的前提下，提高 Vue 代码的类型安全、复用度和可维护性。

主要交付目标为微信小程序，H5 保持基础兼容。

## 2. 设计方向

整体采用“清晰操作 60% / 校友记忆 40%”的平衡方向。

- 清晰操作：深绿品牌色、轻量卡片、明确按钮、稳定的信息层级和克制动效。
- 校友记忆：适量宋体标题、暖纸色内容区域、教育与工作经历时间轴。
- 不使用抒情标语、印章、档案编号和装饰性文字动画。
- 文案直接描述功能、状态和下一步操作。

### 2.1 视觉变量

| 角色 | 色值 | 用途 |
| --- | --- | --- |
| 品牌深绿 | `#153F32` | 主按钮、关键标题背景、品牌区域 |
| 交互青柠 | `#94C52F` | 选中状态、进度、时间轴节点、成功反馈 |
| 页面底色 | `#F6F8F6` | 页面背景 |
| 记忆暖纸 | `#F2EDE2` | 资讯摘要、少量内容强调区域 |
| 分隔灰绿 | `#D6DDD8` | 边框、分隔线、禁用状态 |
| 正文深色 | `#173229` | 正文和标题 |
| 次级文字 | `#718078` | 辅助信息、时间、提示 |
| 错误色 | `#C74B42` | 错误、删除、报名失败 |

全局样式通过 `theme.scss` 管理 SCSS 变量与 CSS 自定义属性，页面不得重复声明品牌色、圆角、阴影和基础间距。

### 2.2 字体与排版

- 页面标题、内容标题和经历模块标题使用系统宋体栈，体现适量校友记忆感。
- 正文、表单、按钮和状态信息使用系统黑体栈，保证小程序可读性和性能。
- 不引入网络字体。
- 页面标题、模块标题、正文、辅助文字形成固定字号层级。

### 2.3 形状与层级

- 页面卡片使用统一中等圆角，避免每个页面自行定义不同半径。
- 阴影只用于主要浮层和高层级卡片，常规列表优先使用底色和分隔线。
- 主按钮为深绿实心；次按钮为浅底或描边；危险操作使用错误色文字或弱底色。
- 表单控件保证微信小程序触控面积，并提供聚焦、禁用、错误和提交中状态。

### 2.4 动效

- 页面主体进入：透明度与小幅上移，时长 220ms。
- 卡片按下：轻微缩放，释放后恢复，时长 180ms。
- 弹层：遮罩淡入，内容由底部或中心短距离进入，时长 220–240ms。
- 签到成功：使用一次“轨迹抵达”动画强化结果，随后稳定显示成功状态。
- 骨架屏使用低频明暗变化，不使用高频闪烁。
- 尊重 `prefers-reduced-motion`；减少动态效果时仅保留必要的透明度反馈。

## 3. 页面设计

### 3.1 登录与注册

- 使用统一品牌头部与表单卡片。
- 验证码登录和密码登录使用分段切换。
- 验证码按钮显示可发送、倒计时和禁用状态。
- 主操作只保留“登录”或“完成注册”。
- “受邀嘉宾直接签到”作为低层级文字入口。
- 手机号、验证码、密码错误在对应字段附近说明，不只依赖 Toast。

### 3.2 资讯

- 使用统一页面标题和暖纸色空状态。
- 当前后端没有资讯接口，因此不展示虚构内容。
- 空状态明确说明“暂无资讯”，不提供无效入口。
- 未来接入资讯接口时复用内容卡片，不改变页面结构。

### 3.3 活动列表

- 首屏展示页面标题和活动列表，不依赖虚构推荐活动。
- 活动卡片包含封面、名称、时间、地点、主办方和报名状态。
- 报名状态由 `registerStart`、`registerEnd`、`limit` 和 `status` 计算。
- 下拉刷新保留，分页加载使用明确的加载、结束和失败反馈。
- 封面缺失时使用品牌占位图，避免拉伸和跳动。

### 3.4 活动详情与报名

- 顶部显示活动封面和简洁日期标识。
- 时间、地点、主办方、剩余名额使用统一元信息组件。
- `exactLocation` 按后端契约作为 JSON 字符串安全解析；解析失败时回退到 `location`。
- 描述和联系方式保持可读段落，不添加装饰性文本。
- 底部固定报名按钮根据未开始、可报名、已满、已截止和提交中状态变化。
- 报名弹层支持多位参与者，逐项校验姓名和手机号，并阻止重复提交。

### 3.5 签到与签到管理

- 签到页突出活动名称、参与者信息和单一签到操作。
- 成功后显示稳定结果页，并执行一次“轨迹抵达”动画。
- 失败时说明失败原因和可执行操作。
- 签到管理页显示已签到人数、总人数、搜索、刷新和添加参与者。
- 搜索、已签到和未签到状态使用统一列表样式。

### 3.6 个人中心

- 顶部展示姓名、头像占位和资料完整度。
- 基础信息、教育经历、工作经历拆分为独立模块。
- 教育与工作经历使用轻量时间轴展示校友成长信息。
- 每个模块统一采用“编辑 / 保存 / 取消”，取消时恢复编辑前数据。
- 删除经历必须提供确认，保存中禁止重复操作。
- 家乡教育和在沪教育保留独立分组。

## 4. 组件与代码架构

### 4.1 基础组件

- `AppLayout`：安全区、背景、底部留白和页面进入效果。
- `PageHeader`：页面标题、说明和可选操作。
- `SurfaceCard`：统一卡片层级。
- `AppButton`：主、次、文字、危险和加载状态。
- `FormField`：标签、输入、帮助和错误状态。
- `StatusTag`：活动、签到和通用状态。
- `EmptyState`：空数据、错误与重试。
- `SkeletonBlock`：加载占位。
- `AppModal`：遮罩、焦点、关闭和操作区。

### 4.2 业务组件

- `ActivityCard`
- `ActivityMeta`
- `RegistrationForm`
- `CheckInStatus`
- `ExperienceTimeline`
- `ExperienceEditor`

组件使用 Vue 3 Composition API、`<script setup lang="ts">`、类型化 `defineProps` 和 `defineEmits`。模板统一使用 UniApp 支持的 `view`、`text`、`button`、`input` 等组件，不混用原生 `div`。

### 4.3 组合式逻辑

- `useAuth`：登录状态、令牌读取和登录失效处理。
- `useCountdown`：验证码倒计时和卸载清理。
- `useAsyncAction`：提交中、错误和重复提交保护。
- `useEditableList`：经历编辑、备份、恢复、添加和删除。

不需要深层响应的状态优先使用 `shallowRef`。派生状态使用 `computed`，避免用 `watch` 同步可计算值。列表编辑需要副本时使用明确的类型化复制函数，不使用散落的 `JSON.parse(JSON.stringify(...))`。

## 5. 后端契约

后端生成 DTO 和路由是本次实现的契约来源。

### 5.1 用户

- 注册：`POST /user/sign_up`
- 登录：`POST /user/sign_in`
- 用户信息：`GET /user/info`
- 更新基础信息：`POST /user/update_info`
- 更新教育经历：`POST /user/update_edu`
- 更新工作经历：`POST /user/update_employment`

教育经历字段必须使用后端定义的 `phase`，修复当前前端的 `phrase`。

### 5.2 活动与签到

- 活动列表：`POST /activity/get_many`
- 活动详情：`POST /activity/get`
- 活动报名：`POST /activity/register`
- 签到：`POST /activity/check_in`
- 报名与签到情况：`POST /activity/get_register`

活动数据继续使用 `cover`、`name`、`location`、`exactLocation`、`sponsor`、`start`、`registerStart`、`registerEnd`、`description`、`contact`、`limit` 和 `status`。

### 5.3 请求层

- 请求层统一添加 `Authorization`，公开接口允许无令牌请求。
- 统一解析 HTTP 错误和后端业务响应。
- 登录失效时清除本地用户信息并重新进入登录页。
- 网络错误保留当前页面状态并提供重试，不直接清空已有数据。
- 不在页面中重复组装鉴权请求头。

## 6. 状态与异常

所有数据页面必须覆盖：

- 初始加载：骨架屏。
- 空数据：说明当前无内容，不显示错误语气。
- 网络错误：说明加载失败并提供重试。
- 提交中：禁用重复操作并显示进行中状态。
- 提交成功：按钮、页面状态和 Toast 文案保持一致。
- 登录失效：统一重新登录。
- 图片失败：品牌占位图。
- 字段缺失：使用“未填写”或隐藏非必要行，不能展示 `undefined`。

## 7. 无障碍与小程序约束

- 交互目标满足触控面积要求。
- 文字和背景保持足够对比度。
- 禁用状态不能只依赖透明度表达。
- 图片提供合理的替代含义或装饰性处理。
- 动效不阻塞操作，并支持减少动态效果。
- 页面适配状态栏、安全区和底部 TabBar。

## 8. 验证

### 8.1 静态验证

- `npm run type-check`
- 微信小程序构建
- 检查模板组件兼容性和 SCSS 变量引用

### 8.2 关键流程

1. 验证码登录、密码登录和注册。
2. 活动列表刷新、分页、空状态和错误重试。
3. 活动详情加载、报名状态和多人报名。
4. 嘉宾签到、签到成功和重复签到反馈。
5. 签到管理搜索、添加参与者和刷新。
6. 基础信息、教育经历和工作经历的编辑、取消和保存。
7. 登录失效后的统一跳转。

### 8.3 视觉验证

- 微信小程序常见屏幕宽度下检查间距、换行和安全区。
- 检查长活动名称、无封面、无主办方、无限名额和已截止状态。
- 检查减少动态效果配置。

## 9. 非目标

- 不修改后端接口或数据模型。
- 不为资讯页面添加虚构数据或本地内容系统。
- 不引入外部 UI 框架或网络字体。
- 不重做后台管理端。
- 不增加与本次视觉和体验目标无关的业务功能。
